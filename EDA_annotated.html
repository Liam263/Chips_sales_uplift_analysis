QVI Chip Sales — Exploratory Data Analysis¶

This notebook performs an end-to-end exploratory data analysis (EDA) on QVI chip sales data. The goal is to understand customer purchasing behaviour across different lifestages and customer segments, and to identify key insights that can inform marketing and product strategy.

Datasets used:

  • QVI_transaction_data.xlsx — individual chip purchase transactions
  • QVI_purchase_behaviour.csv — customer loyalty card profile data (lifestage, premium tier)

Analysis outline:

  1. Data loading & initial inspection
  2. Data cleaning & feature engineering
  3. Outlier detection & removal
  4. Time-series transaction analysis
  5. Pack size & brand feature extraction
  6. Customer dataset exploration & merging
  7. Segment-level sales analysis
  8. Statistical price testing
  9. Target segment deep-dive (brand & pack affinity)

9.2 Average Units Purchased per Customer¶

Calculate how many chip units each customer buys on average, segmented by lifestage and premium tier. This helps distinguish high-volume segments from high-value ones.

In [28]:
# Total units per segment
units_by_segment = (
    merge.groupby(["LIFESTAGE", "PREMIUM_CUSTOMER"])["PROD_QTY"]
    .sum()
    .reset_index(name="total_units")
)

# Number of customers per segment
customers_by_segment = (
    merge.groupby(["LIFESTAGE", "PREMIUM_CUSTOMER"])["LYLTY_CARD_NBR"]
    .nunique()
    .reset_index(name="num_customers")
)

# Merge to compute average units per customer
avg_units = units_by_segment.merge(
    customers_by_segment,
    on=["LIFESTAGE", "PREMIUM_CUSTOMER"]
)

avg_units["avg_units_per_customer"] = avg_units["total_units"] / avg_units["num_customers"]

print(avg_units)

# Pivot for plotting
pivot_avg_units = avg_units.pivot(
    index="LIFESTAGE",

9.4 Statistical Price Testing¶

Use Welch's t-tests to determine whether observed price-per-unit differences between segments are statistically significant:

  • Test 1: Mainstream vs Premium customers
  • Test 2: Midage Singles/Couples vs Young Singles/Couples

from scipy import stats

# Create average price per unit at transaction level (if not already present)
merge["price_per_unit"] = merge["TOT_SALES"] / merge["PROD_QTY"]

# -----------------------------
# 1. Mainstream vs Premium (excluding Budget to keep clean comparison)
# -----------------------------

mainstream = merge[merge["PREMIUM_CUSTOMER"] == "Mainstream"]["price_per_unit"]
premium = merge[merge["PREMIUM_CUSTOMER"] == "Premium"]["price_per_unit"]

t_stat1, p_val1 = stats.ttest_ind(mainstream, premium, equal_var=False)
